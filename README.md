# Clinical Trial Analytics Dashboard

An end-to-end clinical data analytics project built using simulated Phase III trial data following **CDISC ADaM conventions** and **CTCAE v5.0** adverse event grading terminology.

The objective is to demonstrate a complete clinical data workflow — from structured ADaM datasets through Python-based analysis to an executive-level interactive dashboard.

**Live demo:** [namratharao22.github.io/clinical-trial-dashboard](https://namratharao22.github.io/clinical-trial-dashboard)

---

## Project overview

This project answers four clinical research questions:

**Research Question 1**  
Which trial arm demonstrates stronger enrollment velocity and patient retention?

**Research Question 2**  
What is the adverse event profile by System Organ Class and CTCAE grade?

**Research Question 3**  
Which sites are highest enrolling and which have the highest dropout rates?

**Research Question 4**  
What is the TFL production and validation status across Tables, Figures, and Listings?

## Dataset descriptions

### ADSL.csv — Subject-Level Analysis Dataset
CDISC ADaM ADSL-style dataset. One row per subject.

| Column | Description |
|---|---|
| USUBJID | Unique subject identifier |
| SITE | Enrolling site name |
| ARM | Treatment or Placebo |
| AGE | Age at enrollment |
| SEX | M / F |
| RACE | Race category |
| ENROLL_MONTH | Month of enrollment (1–6) |
| STATUS | Completed / Active / Discontinued / Screening Failure |
| PROTOCOL | Protocol identifier |
| PHASE | Trial phase |

### ADAE.csv — Adverse Event Analysis Dataset
CDISC ADaM ADAE-style dataset. One row per adverse event.

| Column | Description |
|---|---|
| AEID | Unique AE identifier |
| USUBJID | Subject identifier (links to ADSL) |
| ARM | Treatment or Placebo |
| SOC | System Organ Class (MedDRA) |
| AETERM | Adverse event preferred term |
| CTCAE_GRADE | Grade 1–4 per CTCAE v5.0 |
| SERIOUS | Y / N (Grade 3+ flagged as serious) |
| ONSET_WEEK | Week of AE onset from first dose |
| RESOLVED | Y / N |

### TFL_Tracker.csv — TFL Production Status
One row per TFL deliverable (Table, Figure, or Listing).

| Column | Description |
|---|---|
| TFL_ID | Unique TFL identifier |
| TYPE | Table / Figure / Listing |
| TITLE | Descriptive TFL title |
| PROGRAM | SAS program filename |
| STATUS | Validated / In QC Review / Pending |
| VALIDATED_BY | Initials of validator |
| QC_BY | QC programmer |
| DOMAIN | Source ADaM domain (ADAE, ADSL, ADLB, ADEFF, ADPK) |
